채팅 시스템 프로젝트

* 기본시나리오 : 로그인 -> 채팅로비 입장 -> 현재 생성된 채팅룸 리스트 실시간 갱신 -> 채팅룸 입장 -> 채팅
* 구조 : typical 3-tier - 클라이언트 <-> 프론트엔드서버 <-> 백엔드서버(레디스)

-------------
클라이언트

1. 사용플랫폼 : 윈도우즈 환경 콘솔 이상. 안드로이드 에뮬 이상도 가능
2. 3종 제작 :
 a. 채팅 클라이언트
 b. 더미 클라이언트
    - 정해진 프로토콜에 따라 채팅 룸까지 들어와서 무작위 개수의 현재 시간 정보를 출력하고 5~10초 사이 기다린 후 로그아웃
 c. 현재 채팅 시스템 상황 보여주는 모니터링 클라이언트
- 채팅 룸 개수, FE 서버별 사용자 수(더미 클라이언트 포함), 채팅메시지 수 작성 TOP10을 실시간으로 보여줌(더미 제외)

-----------
FE 서버

1. 두 대 이상 사용되야 함 - 서버간 채팅 룸 목록 동기화
2. 클라이언트와의 채팅 메시지 전달 - 한 채팅룸에 접속한 사용자가 각기 다른 서버에 접속한 경우 두가지 방법중 하나 선택
 a. 동일 서버로 접속시킴 - 새로운 서버로 접속을 열고 기존 서버와 접속 끊음
 b. 서버간 메시지 릴레이

-----------
BE 서버

1. 유저 정보 저장 : 로그인, 실제 사용자/더미 사용자 구별, 채팅 메시지 개수 저장
2. 모니터링 클라이언트와 연결해서 현 시스템 상황 정보 전달
